## 개발 목적

현재 Codex Prompt Skill Factory의 후보 탐지/승인/생성 흐름에 **공용 Prompt Enrichment / Skill Spec Compiler** 단계를 추가한다. 목표는 특정 예시별 좋은 프롬프트를 하드코딩하는 것이 아니라, 모든 Skill 후보를 `목표/입력/맥락/제약/절차/검증/출력` 계약으로 일반화하고, 부족한 항목을 진단해 더 좋은 프롬프트 템플릿·질문·품질 체크리스트를 자동 생성하는 것이다.

## 전문가 설계 판단

- 이번 개발의 핵심 산출물은 `better prompt` 문장 목록이 아니라, 모든 후보에 공통 적용 가능한 **Task/Prompt/Skill Contract**다.
- 후보 예시는 evidence로만 사용하고, 생성되는 Skill 본문에는 특정 파일명·날짜·브랜치·고객명을 고정하지 않는다.
- enrichment는 기본적으로 deterministic local rule 기반으로 구현한다. 그래야 테스트 가능하고 민감정보 유출 위험이 없다.
- 품질 점수는 승인 자동화가 아니라 사람의 판단을 돕는 보조 신호로 사용한다.
- 기존 `rules.py`, `similarity.py`, `analytics.py`, `approvals.py`, `dashboard.py` 책임을 침범하지 않고 새 모듈을 추가해 통합한다.

## 잔여 리스크 / 후속 과제

- 현재 enrichment는 deterministic rule 기반이다. 자연어 품질을 더 높이려면 선택형 LLM enrichment backend를 별도 옵션으로 추가할 수 있다.
- variable slot 추출은 파일/URL/명령/날짜/브랜치 중심이다. 고객명, 제품명, 도메인 특화 엔티티 추출은 후속 개선 대상이다.
- `review`는 터미널 기반 인터랙션이며, 대시보드에서 직접 승인/생성하려면 로컬 서버 또는 VS Code Extension이 필요하다.
