> **현재 기준 고정:** 이 문서는 당시 구현/계획 기록이지만, 프로젝트의 현행 최종 목적은 **Codex Prompt Skill Factory**를 다음 제품으로 완성하는 것이다: 누구나 설치해서 Codex 프롬프트 사용 패턴을 로컬에서 수집·분석하고, 반복 요청을 Skill 후보로 제안하며, 사용자가 승인한 후보를 Codex가 바로 사용할 수 있는 Skill로 자동 생성·설치하는 installable CLI 제품. 현행 범위는 `docs/codex_prompt_skill_factory_dev_doc.md`를 우선한다.

이 문서는 이번 개발의 범위를 고정하고, 구현이 목적 밖으로 확장되지 않도록 하기 위한 작업 문서다.

## 개발 목적

Codex Prompt Skill Factory의 최종 목표를 “누구나 설치해 자신의 Codex 프롬프트 사용 패턴을 로컬에서 분석하고, 승인한 Skill을 자동 생성·설치해 이후 작업에 바로 활용하는 제품형 CLI”로 명확히 재정의한다. 현재 문서의 repo-local PoC 중심 서술을 제품 목표, UX, 설치 흐름, 검증 가능한 시나리오 테스트 중심 문서로 재작성한다.

## 개발 범위

- `docs/codex_prompt_skill_factory_dev_doc.md`를 제품형 개발 문서로 재작성
- 최종 목표, 사용자, 핵심 가치, 제품 범위, 제외 범위 명확화
- 설치형 CLI 기준 명령 체계 정의: `init`, `inbox`, `promote`, `dashboard`, `doctor`
- 내부 고급 명령과 제품형 기본 명령의 관계 정리
- 현재 구현 자산과 목표 제품 사이의 gap 분석 작성
- 실제 검증 가능한 E2E/시나리오 테스트 목록 작성
- 제품 완료 기준과 출시 전 체크리스트 정의

## 제외 범위

- 이번 작업에서 코드 구현 변경은 하지 않는다.
- 새 기능 아이디어를 추가 확장하지 않는다.
- 외부 LLM/API, 서버, VS Code Extension, 원격 동기화는 포함하지 않는다.
- 기존 테스트 파일 수정 또는 실행은 이번 문서화 작업 범위가 아니다.
