## 개발 목적

중복 프롬프트에서 생성되는 Skill 후보가 실제 사용 가능한 Skill이 되도록, 특히 유사도 기반 후보가 “유사 프롬프트 클러스터를 Skill로 만들라”는 메타 Skill로 생성되는 문제를 제거한다. 결과적으로 `inbox → promote` 흐름에서 사용자가 승인할 만한 구체적 Skill 초안과 목적별 출력 형식을 제공한다.

## 개발 범위

- 유사도 기반 후보의 목적/제목/slug/goal/workflow를 로컬·결정론적으로 구체화한다.
- 생성 `SKILL.md`의 `Output format`을 task archetype/domain별 output contract에 맞게 렌더링한다.
- Skill 설치 추천 가능성을 판단할 수 있는 install readiness 등급을 품질 정보에 추가한다.
- 위 변경을 검증하는 단위 테스트와 실제 렌더링 테스트를 추가한다.
- 변경된 생성 사양을 `docs/skill_template_spec.md`와 현행 개발 문서에 동기화한다.

## 제외 범위

- 외부 LLM/API 기반 후보 합성은 추가하지 않는다.
- VS Code Extension, 웹 서버, 원격 대시보드는 만들지 않는다.
- 특정 테스트 예시 하나만 위한 하드코딩 Skill을 만들지 않는다.
- `.codex/skills/`에 생성 Skill을 커밋하지 않는다.
- 기존 Golden Path 명령 체계를 바꾸지 않는다.
