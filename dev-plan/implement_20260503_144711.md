## 개발 목적

Codex Prompt Skill Factory의 고정 목적인 “설치형 CLI가 Codex 프롬프트를 로컬에서 안전하게 수집·분석하고, 반복 요청을 Skill 후보로 제안·승인·생성하는 제품”을 지키기 위해, 전문가 리뷰에서 실제 제품 목표에 직접 영향을 주는 5개 안정화 항목을 구현한다.

## 개발 범위

- legacy `.codex/hooks/log_*.py`를 활성 CLI hook handler로 위임하는 안전 wrapper로 단일화한다.
- `init`이 기존 `hooks.json`을 덮어쓰지 않고 merge/backup하도록 개선한다.
- `hook_handlers.py`의 Stop hook text signal 계산 전에 secret redaction을 방어적으로 적용한다.
- `inbox`가 non-TTY/CI 환경에서 interactive prompt로 block되지 않도록 자동 비대화형 전환한다.
- 위 항목을 검증하는 회귀 테스트를 추가한다.

## 제외 범위

- Codex 자체 런타임 수정은 하지 않는다.
- VS Code Extension, 웹 서버, 원격 동기화는 만들지 않는다.
- user custom rules, uninstall, rotate 같은 기능 확장은 이번 범위에 포함하지 않는다.
- 기존 Golden Path 명령 이름과 기본 흐름은 바꾸지 않는다.
- 기존 미커밋 변경을 되돌리지 않는다.

## 전문가 리뷰 기준

- Local-first: 로그/프롬프트를 외부로 보내지 않는다.
- No secret retention: hook 경로가 legacy/direct/CLI 어느 쪽이든 secret 저장 위험을 줄인다.
- Product UX first: `init → inbox → promote → doctor/dashboard` 흐름을 막지 않는다.
- User settings safe: 사용자의 기존 Codex hook 설정을 보존한다.
- Regression protected: 보안/설정/UX 변경은 테스트로 고정한다.
